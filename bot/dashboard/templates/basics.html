Cybersecurity Basics

Core concepts used throughout ARGUS

CVE

Common Vulnerabilities and Exposures. A unique identifier assigned to a publicly disclosed security vulnerability. Each CVE represents a specific security flaw that affects software, hardware, firmware, operating systems, network devices, cloud services, or applications.

Example

  • CVE-2024-4115
  • CVE-2025-12345

Each identifier refers to one specific vulnerability.

Purpose
CVEs provide a standardized naming system that allows security teams, vendors, researchers, and tools to reference the same vulnerability consistently.

Without CVEs

  • Vendor A: "Authentication Bypass Bug"
  • Vendor B: "Remote Acess Vulnerability"
  • Researcher: "Router Login Flaw"

With CVEs

  • CVE-2024-4115

Everyone references the same vulnerability.

CVSS

Common Vulnerability Scoring System. Measures the technical severity of a vulnerability on a scale from 0.0 to 10.0.

  • None: 0.0
  • Low: 0.1 - 3.9
  • Medium: 4.0 - 6.9
  • High: 7.0 - 8.9
  • Critical: 9.0 - 10.0

Example:
CVSS evaluates factors such as:

  • Attack complexity
  • Required privileges
  • User interaction
  • Impact on confidentiality
  • Impact on integrity
  • Impact on availability

Limitations
CVSS only measures technical severity. It does not indicate:

  • Whether attackers are exploiting it
  • Whether exploit code exists
  • Whether your organization is affected

CWE

Common Weakness Enumeration. Describes the root cause of a vulnerability. While CVE identifies a specific vulnerability, CWE identifies the category of weakness.

Example:
Vulnerability

  • CVE-2024-4115

Underlying Weakness

  • CWE-79
Cross-Site Scripting (XSS).

Why CWE Matters
CWE helps organizations identify recurring security weaknesses.

Example:
50 vulnerabilities, 40 of them are CWE-79
This indicates a systemic XSS problem rather than isolated issues.

KEV

Known Exploited Vulnerabilities. A catalog maintained by the Cybersecurity and Infrastructure Security Agency (CISA) containing vulnerabilities that are confirmed to be actively exploited in real-world attacks.

Why KEV Matters
Many vulnerabilities have:

  • High CVSS
  • No exploitation

However, KEV vulnerabilities have:

  • Real attackers
  • Real incidents
  • Real exploitation

Example
CVE-2021-44228 (Log4Shell)
CVSS: 10.0
KEV: Yes

Risk Implications
If a vulnerability appears in KEV:

  • Exploit exists
  • Attackers are using it
  • Immediate remediation recommended

EPSS

Exploit Prediction Scoring System. Estimates the probability that a vulnerability will be exploited in the next 30 days. Developed by the Forum of Incident Response and Security Teams (FIRST), EPSS uses machine learning and real-world threat intelligence to predict exploitation likelihood on a scale from 0.0 to 1.0.

Score
Example:
EPSS = 0.95
Means a very high likelihood of exploitation.

Percentile
Example:

EPSS 0.56%
Percentile P99.8

Means a higher exploitation probability than 99.8% of all tracked vulnerabilities and among the vulnerability most likely to be exploited.

Why EPSS Matters
CVSS asks: How severe is the vulnerability
EPSS asks: How likely is exploitation

Example

CVE CVSS EPSS
CVE-A 9.8 0.01%
CVE-B 6.5 2.50%

Although CVE-A is technically more severe, CVE-B may be more likely to be attacked.

ARGUS Risk Score

ARGUS Risk Score is a custom prioritization metric designed to help security teams focus on vulnerabilities that pose the greatest risk to their environment.

Unlike CVSS, which measures only technical severity, ARGUS combines multiple risk factors to better reflect real-world exposure and business impact.

Factor Description
CVSS Technical severity of the vulnerability.
KEV Additional weight if active exploitation has been confirmed by CISA.
Asset Priority Business importance of the affected asset.
EPSS Likelihood of exploitation in the next 30 days.

Example

Metric Value
CVSS 9.8
KEV Yes
Priority Critical
EPSS 99.8%
ARGUS Risk Score 1174

Higher Risk Scores indicate vulnerabilities that should be prioritized for investigation and remediation.
